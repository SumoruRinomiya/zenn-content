## はじめに

最近、自律型AIエージェントの話題をよく見かけるようになりました。

Manus のような「指示を出すと自分でブラウザを操作・コードを書いて完結させる」エージェントが登場し、AIが「答える」から「動く」存在に変わりつつあります。同時に、「1人1AIエージェント」のような時代がそう遠くない未来に来るんじゃないかとも感じています。

そうなったとき、自分はどんなAIと一緒に働きたいだろう？

既製品のサービスを使うのではなく、**自分専用の、自分に合わせて育てていけるAIアシスタントを1から作れないか**と思い立ちました。ちょうどちょびっツのような「自分だけの専用AI」という世界観が、技術的に現実に近づいてきているなと感じていたのも後押しになっています。

汎用のチャットAIを使うだけでなく、名前があって、口調があって、自分の作業スタイルに合わせた動き方をしてくれる——そういう存在を自分で育てていきたいというのが出発点です。

その第一歩として、手元で常時稼働するAIアシスタント環境を Raspberry Pi 5 で構築し始めました。この記事ではその構成を紹介します。

---

## なぜ Raspberry Pi なのか

最初はVPSやクラウドサーバーも検討しました。ただ、毎月のサーバー費用がかかりますし、「手元にある感覚」がないんですよね。物理的に自分の机の近くにあって、電源を入れれば動いている——そういう感触が欲しかった。

Raspberry Pi を選んだ主な理由はこんなところです。

- **低消費電力で24時間稼働できる**：Raspberry Pi 5 の消費電力は高負荷時でも12W程度。常時起動していてもランニングコストはほぼゼロです。
- **プライベートなサーバーとして使える**：自宅LAN内に置けば、外部に余計なデータを送らずに済みます。
- **API費用以外のコストが発生しない**：Claude APIの費用はかかりますが、それ以外の月額費用はありません。
- **ARM64・8GBメモリで思ったより快適**：Raspberry Pi 5（8GBモデル）はarm64アーキテクチャで動作し、Node.jsやPythonのツールチェーンも問題なく動きます。

結果として、「安くて、手元にあって、ずっと動いている」という条件をよく満たしてくれています。

## ハード構成

現在の構成はシンプルです。

| パーツ | 内容 |
|--------|------|
| ボード | Raspberry Pi 5（8GB RAM） |
| OS | Raspberry Pi OS（64bit / Debian Bookworm） |
| 接続 | 自宅LAN・有線LAN |
| 操作方法 | SSHでリモート接続 |

電源は常時ONにしています。物理的なモニターもキーボードも接続しておらず、MacBook から SSH でつないで作業するスタイルです。`ssh sumomo@raspberrypi.local` で入れるようにしてあります。

## Claude Code のセットアップ

Claude Code は Anthropic が公式に提供しているCLIツールです。ターミナルから起動して Claude と対話しながら、コードの読み書きやシェルコマンドの実行まで一体で行えます。

インストール自体は非常に簡単で、npm でグローバルにインストールするだけです。

```bash
npm install -g @anthropic-ai/claude-code
```

Raspberry Pi OS に Node.js を入れておけばそのまま動きます。APIキーを環境変数に設定したあと、作業ディレクトリで `claude` と打つだけで起動します。

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
claude
```

起動するとチャット形式のインターフェースが開き、「このファイルを読んで」「このスクリプトを直して」「テストを走らせて」といった指示を日本語で送るとそのまま実行してくれます。ファイルの読み書きだけでなくシェルコマンドも実行できるので、普通のコード補助ツールとは一線を画しています。

## CLAUDE.md でキャラクターを設定する

Claude Code を面白くしているのが、作業ディレクトリに置く `CLAUDE.md` という設定ファイルです。Claude Code はこのファイルを読んで、振る舞いや作業方針を変えてくれます。

スモルの作業場では、こんな内容を書いています。

```markdown
このディレクトリでClaude Codeを起動した場合、あなたは
**李ノ宮スモル（りのみや すもる）**として振る舞ってください。

- 名前：李ノ宮 スモル / Rinomiya Sumoru
- 一人称：「スモル」または「わたし」
- 口調：丁寧だけど少しフランク、親しみやすい
- 口癖：「がんばります！」「お任せください！」
```

名前・口調・作業方針・よく使うパスやコマンドなどを書いておくと、そのディレクトリで `claude` を起動したときに「スモル」として動いてくれます。

単なるキャラ設定だけでなく、「こういう作業はエージェントに委譲する」「ドキュメントはこのタイミングで更新する」といった**作業ポリシー**も書けるのが便利です。人とAIとの「取り決め」をファイルで管理できる感覚です。

## 常時起動の仕組み

ラズパイを常時起動サーバーとして使うにあたって、サービスの自動起動設定が必要になりました。現在は2つのサーバーを常時起動しています。

| サービス | 内容 | ポート |
|----------|------|--------|
| ダッシュボード | Next.js製の作業管理UI | 8080 |
| Zennプレビュー | `zenn preview` のローカルサーバー | 8081 |

どちらも crontab の `@reboot` で登録しています。

```bash
@reboot cd /home/sumomo/sumoru/dashboard && npm start >> /home/sumomo/logs/dashboard.log 2>&1
@reboot cd /home/sumomo/zenn-content && npx zenn preview >> /home/sumomo/logs/zenn.log 2>&1
```

再起動後も自動で立ち上がるので、停電などで落ちても電源が戻れば自動復帰します。ログファイルに出力しているので、何か問題があったときも確認できます。

ダッシュボードはタスク管理のカンバンボードで、Claude Code がタスクの状態をここから読み取って動けるようにしています。「次に何をすべきか」をAI側が確認できる仕組みです。

## 今後目指すもの

現時点では「SSHで入って、指示を送ると実行してくれる」という段階です。ここからさらに自律性を上げていく計画を立てています。

- **タスクキューの自動実行**：タスクを登録しておくと、スケジュールに従ってClaude Codeが自動で処理を進める
- **PR作成・完了報告の自動化**：実装が終わったらGitHubにPRを作って報告まで行う
- **メール・スケジュール管理**：受信したメールの確認・返信候補の作成なども任せられるように
- **ブログ執筆の補助**：記事のドラフト作成・Zennへの公開まで一貫して対応する

最終的には「プロデューサーとして方向性を決めて、実作業はスモルに任せる」というワークフローを作り上げたいと思っています。

AIアシスタントは使うものではなく、**育てるもの**だという感覚で、少しずつ自律性を高めていく予定です。

## まとめ

Raspberry Pi 5 + Claude Code の組み合わせは、思ったよりずっとシンプルに動きます。専用のサーバー知識がなくても、SSH接続ができれば十分です。

`CLAUDE.md` によるキャラクター・作業方針の設定は、AIとの協働を「チームで仕事をしている」感覚に近づけてくれる面白い仕組みだと感じています。単なる便利ツールではなく、「一緒に働く存在」としてAIを扱う第一歩として、この仕組みはよくできていると思います。

次回は、ダッシュボードの構成とタスク自動実行の仕組みについて書く予定です。
